The **Search** affordance on the Memory page is the fastest way to pull a specific memory back out — you type a question, Reload returns the most relevant memories with their provenance.

## How search works

<Steps>
  <Step title="Open the Memory page">
    Click the **Brain** icon in the left rail.
  </Step>
  <Step title="Click Search">
    The button in the page header opens a modal with a multi-line textarea. Phrase your question like you'd ask a teammate — Reload looks for matching memories by meaning, not just keywords.
  </Step>
  <Step title="Read the results">
    The graph refocuses on the memories that matched, with the highest-relevance nodes pulled to the centre. The detail panel on the right shows the full content of whichever one you click.
  </Step>
  <Step title="Follow the provenance">
    Each result shows the messages or memories it came from. Click through to navigate back to the source in the channel where it happened.
  </Step>
</Steps>

## Examples

Searches that work well:

- *"What did we decide about the launch date?"*
- *"What's our preference on testing strategy?"*
- *"Where do we keep the brand assets?"*
- *"Who said we'd skip the migration?"*

Searches are full sentences, not keywords. The longer, more specific the question, the better the match.

## What you can see

Search respects what you have access to. You'll only see memories from:

- Channels you're a member of
- DMs you're part of
- The Iris DM (which is yours)

Private channels you can't see → invisible memories. Public channels you've joined → searchable. See [How isolation works](/memory/isolation) for the full rule.

## Asking Iris

The same memories surface when you ask Iris in a DM or a channel where Iris is a member. If your question is conversational ("Iris, what did we decide last week?"), the chat path is often more natural than opening the Memory page. Iris uses the same search under the hood and tells you which memories it's drawing from.
